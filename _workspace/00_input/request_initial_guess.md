## 도메인 (합리적 추론)
"기타 코드 연습"이라는 명칭과 일반적인 기타 코드 연습 앱의 표준 기능을 토대로 다음을 핵심 기능으로 가정한다:

### 핵심 기능 (MVP)
1. **코드 다이어그램 표시** — 6현 기타 지판에 코드 운지를 시각화
   - 프렛 번호, 줄별 핑거링, 뮤트 줄(X), 오픈 줄(O), 베이스 음
2. **코드 카탈로그** — 주요 코드 데이터셋 (메이저/마이너/세븐스 등 최소 20개 이상)
3. **코드 선택 / 검색** — 코드명 입력 또는 리스트에서 선택
4. **연습 모드** — 코드를 순서대로 또는 랜덤으로 표시하며 사용자가 다음으로 넘김
   - 타이머(자동 진행) 옵션
   - 진행률/카운트 표시

### 부가 기능 (시간 허용 시)
- 즐겨찾기/연습 세트
- 키별 그룹화 (C 키, G 키 등)
- 다크 모드

## UI 비주얼 (Figma 추론)
디자인 파일 미열람으로 비주얼 디테일 불명. 다음 기본 원칙을 따른다:
- 모바일 우선, 단순/깨끗한 카드형 레이아웃
- 코드 다이어그램이 중심(크고 명확하게)
- 부드러운 그림자, 적당한 코너 라운딩
- 색상: 톤 다운된 메인 컬러 1색 + 중성 회색 (다크/라이트 호환)
- 한국어 UI 텍스트 (제목, 버튼 등)

## 수용 기준 (Acceptance Criteria, 후보)
- [ ] 사용자는 코드 목록을 보고 하나를 선택할 수 있다
- [ ] 선택된 코드는 정확한 운지(프렛/줄) 다이어그램으로 표시된다
- [ ] 코드 데이터는 도메인 모델로 분리되어 있고 단위 테스트로 검증된다
- [ ] 연습 모드에서 "다음 코드"를 표시할 수 있다
- [ ] 빌드(`npm run build`)와 테스트(`npm test`)가 통과한다
- [ ] 컴포넌트와 도메인 로직이 분리되어 있다 (관심사 분리)

## 제약
- 사운드 재생, 마이크 입력, 외부 API 호출 등은 MVP 범위 외
- 백엔드 없음. 모든 데이터는 클라이언트 정적 데이터
- 라이센스 자유로운 데이터만 사용 (코드 운지 정보는 공개 도메인)
